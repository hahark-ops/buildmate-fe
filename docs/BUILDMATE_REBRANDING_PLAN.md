# Buildmate Rebranding Plan

## 목표
기존 커뮤니티 프로젝트를 `빌드메이트`로 리브랜딩해, AI 툴을 활용해 사이드 프로젝트를 만드는 메이커들이 팀원을 찾고 실시간으로 협업을 시작하는 커뮤니티로 전환합니다.

## 브랜드 정의
- 서비스명: 빌드메이트
- 포지셔닝: 팀원 모집 + 실시간 협업 시작
- 핵심 메시지: 혼자 만들다 막히면, 함께 만들 메이트를 찾으세요.

## 사용자 정의
- AI 툴을 활용해 사이드 프로젝트를 만드는 사람
- 바이브 코딩 입문자
- 기획은 있는데 같이 만들 사람을 찾는 메이커
- 프론트/백엔드/디자인 역할을 나눠 협업하고 싶은 소규모 팀

## 현재 기능을 Buildmate 문맥으로 바꾸는 방식
- 게시글: 프로젝트 모집글
- 댓글: 질문 / 관심 표현
- 좋아요: 관심 표시
- DM: 협업 채팅
- unread/read: 협업 메시지 상태 확인
- Web Push: 부재 중 협업 알림 구조

## 구현 우선순위
### 1단계: 텍스트 리브랜딩
- 로고와 서비스명 교체
- 메인 카피 교체
- 로그인/회원가입/프로필/DM 문구 교체
- 게시글 작성 페이지를 프로젝트 모집 문맥으로 수정

### 2단계: 시각 리브랜딩
- 색상/타이포/아이콘 정리
- 메인 페이지 첫인상 변경
- 카드, 버튼, 빈 상태, 배지 문구 정리

### 3단계: README/영상 정리
- README 소개를 Buildmate 기준으로 재작성
- 영상 데모 흐름을 팀 모집 -> 댓글 -> DM -> unread/read 중심으로 재구성

## 저장소별 작업 범위
### Frontend
- 헤더 로고/페이지 타이틀/버튼 카피
- 메인 페이지 설명 문구
- 게시글 상세/작성 화면 문맥
- DM 화면 상태 문구
- 프로필/회원가입/로그인 텍스트

### Backend
- README/문서/보고서 설명
- 응답 메시지 중 사용자 노출 텍스트가 있으면 Buildmate 문맥으로 정리
- 배포/아키텍처 설명은 기존 강점을 유지하되 서비스 설명만 교체

## 완료 기준
- 첫 화면만 봐도 원본 과제와 구별됨
- README와 영상 설명이 모두 Buildmate 기준으로 일관됨
- 기능 설명이 `메이커 팀 빌딩 커뮤니티`로 자연스럽게 이어짐
- 기존 기술 강점(DM, Redis, CI/CD, k3s, 테스트코드)이 그대로 살아 있음
