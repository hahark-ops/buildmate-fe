# Buildmate Start Here

이 복사본은 `아무 말 대잔치`를 `빌드메이트`로 리브랜딩하기 위한 작업 시작점입니다.

## 프로젝트 정의
- 프로젝트명: 빌드메이트
- 한 줄 설명: 사이드 프로젝트 팀원을 찾고, 공개 질문과 실시간 협업 채팅으로 협업을 시작할 수 있는 팀 빌딩 서비스
- 제품 성격: 범용 커뮤니티가 아니라 `팀원 모집 + 협업 시작`에 특화된 목적형 서비스
- 초기 타깃: 바이브 코딩 입문자, AI 툴을 활용해 빠르게 MVP를 실험하는 메이커, 첫 MVP를 같이 만들 팀원이 필요한 사용자

## 이번 리브랜딩의 핵심 판단
- `커뮤니티`처럼 보이게 만드는 것이 아니라 `프로젝트 팀원 모집 서비스`처럼 보이게 만드는 것이 목표입니다.
- AI는 서비스의 전체 정체성이 아니라, 이 서비스에 잘 맞는 `초기 사용자군`을 설명하는 관점으로 사용합니다.
- 기능을 새로 많이 붙이기보다, 기존 기능의 의미를 아래처럼 재정의합니다.
  - 게시글: 프로젝트 모집글
  - 댓글: 공개 질문 / 참여 문의
  - 좋아요: 합류 관심 표시
  - DM: 협업 제안과 구체 논의를 시작하는 채널
  - unread/read: 협업 메시지 상태 확인

## 실제 작업 순서
1. `docs/BUILDMATE_IMPLEMENTATION_CHECKLIST.md`를 먼저 읽습니다.
2. 공유 셸(헤더, 로고, 공통 CTA, 프로필 드롭다운) 문자열부터 정리합니다.
3. 메인 / 인증 / 모집글 / DM / 프로필 순서로 페이지별 문맥을 바꿉니다.
4. README와 영상용 설명 문구를 마지막에 맞춥니다.
5. Playwright와 기존 테스트를 카피 변경에 맞게 수정합니다.
